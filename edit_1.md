## 1. 단일 장애점(SPOF) 및 동기식 지연 문제의 심층 해결책

Flask와 같은 동기식(Synchronous) 웹 프레임워크에서 외부 API를 호출하고 기다리는 구조는 서버 자원을 급격히 고갈시킵니다. 타임아웃을 15초로 설정하더라도, 그 15초 동안 해당 워커(Worker) 스레드는 아무것도 하지 못하고 멈춰 있게 됩니다.

### **A. 비동기 작업 큐 (Asynchronous Task Queue) 도입**

* **개념:** 클라이언트가 분석을 요청하면 서버는 즉시 "작업 접수 완료(202 Accepted)" 상태만 반환하고, 실제 스크래핑과 AI 분석은 백그라운드 워커가 처리하는 방식입니다.
* **구현:** Python 환경에서는 **Celery와 Redis(또는 RabbitMQ)** 조합이 표준입니다.
* **장점:** Reddit이나 TikTok 등에서 대량의 데이터를 스크래핑하고 LLM이 분석을 마칠 때까지 프론트엔드가 멈추지 않습니다. 작업이 완료되면 웹소켓(WebSocket)이나 폴링(Polling)으로 결과를 받아오므로 사용자 경험(UX)이 극대화됩니다.

### **B. 서킷 브레이커 (Circuit Breaker) 패턴 적용**

* **개념:** 외부 API(OpenAI 등)에 장애가 발생했을 때 무의미한 재시도(Retry)를 반복하면 서버 리소스만 낭비됩니다. 서킷 브레이커는 오류율이 임계치를 넘으면 '회로를 차단(Open)'하여 외부 API 호출을 즉시 중단합니다.
* **구현:** Python의 `pybreaker` 같은 라이브러리를 활용합니다.
* **장점:** OpenAI 서버가 완전히 다운되었을 때, 타임아웃까지 기다리지 않고 즉시 준비된 Fallback(과거 캐시 데이터 등)을 반환하여 시스템 연쇄 장애(Cascading Failure)를 완벽히 차단합니다.

### **C. 멀티 LLM 프로바이더 라우팅 (Multi-Provider Routing)**

* **개념:** 단일 AI 모델(GPT-4)에만 의존하는 것 자체가 SPOF입니다.
* **구현:** `LiteLLM`과 같은 라우팅 라이브러리를 사용하여 주력 모델(OpenAI) 호출 실패 시, 즉각적으로 대안 모델(Claude 3, Gemini 등)로 요청을 우회(Fallback)시킵니다.
* **장점:** 특정 AI 기업의 서버 장애와 무관하게 마케팅 콘텐츠 생성 및 데이터 분석 파이프라인이 24시간 무중단으로 돌아갑니다.

## 2. 샘플 데이터와 실시간 데이터 혼재의 심층 해결책

단순히 `.env` 파일로 분기하여 JSON 파일을 읽어오는 방식(`if/else`)은 컨트롤러(app.py) 코드의 복잡도를 높이고, 비즈니스 로직과 데이터 접근 로직을 강하게 결합시킵니다.

### **A. 의존성 주입 (Dependency Injection) 및 전략 패턴 (Strategy Pattern)**

* **개념:** 애플리케이션 코드가 "데이터를 어디서 가져오는지" 몰라도 되게 만드는 설계입니다.
* **구현:** 파이썬의 클래스와 인터페이스(ABC)를 활용해 `DataInterface`를 만듭니다. 이를 상속받는 `RealApiDataService`와 `MockDataService` 클래스를 각각 구현합니다. 서버가 구동될 때 환경 변수에 따라 알맞은 클래스를 '주입'해 줍니다.
* **장점:** 엔드포인트 코드는 단 한 줄도 수정할 필요가 없습니다. 새로운 소셜 미디어 플랫폼이나 데이터 소스가 추가되더라도 기존 코드를 건드리지 않고 확장이 가능합니다.

### **B. 네트워크 계층 모킹 (Network-Level Mocking)**

* **개념:** 애플리케이션 내부 로직을 수정하여 가짜 데이터를 반환하는 대신, 애플리케이션이 밖으로 보내는 HTTP 요청 자체를 중간에서 가로채서(Intercept) 가짜 응답을 주는 방식입니다.
* **구현:** 개발 환경에서 `responses` 모듈이나 `WireMock` 같은 도구를 사용해 특정 URL(예: Apify 또는 OpenAI 엔드포인트)로 가는 요청만 낚아채어 로컬 JSON 응답을 반환합니다.
* **장점:** 서버 코드는 완벽하게 "실제 API를 호출하는 상태"를 유지합니다. 코드를 배포할 때 테스트 환경 전용 코드가 섞여 들어갈 위험(Human Error)이 원천 차단됩니다.

### **C. 로컬 캐시 DB 활용 (JSON 파일 탈피)**

* **개념:** 단순 텍스트 파일인 JSON 대신, 로컬 개발 시에는 가벼운 SQLite나 Redis를 활용해 실제 데이터베이스 쿼리와 동일한 환경을 구축합니다.
* **장점:** 트렌드 분석 대시보드에서 날짜별 필터링, 검색, 정렬 같은 기능을 개발할 때, JSON 파일로는 구현하기 어려운 복잡한 쿼리 로직을 실제 환경과 똑같이 테스트할 수 있습니다.
